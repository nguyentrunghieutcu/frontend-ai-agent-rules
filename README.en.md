# Frontend AI Agent Rules - Free

A free rules kit for **Claude Code, Codex, Cursor, Gemini, Windsurf, Continue and GitHub Copilot** when working in frontend codebases built with React, Next.js, Vite and TypeScript.

Goal: help AI coding agents edit code with the right scope, follow existing project patterns, avoid heavy commands, avoid unnecessary dependencies and reduce pull request noise.

## Included

- `CLAUDE.md` - the main guideline file for AI agents.
- `RULE.md` - detailed rules for commands, scope, architecture and code quality.

## Quick setup

Copy these 2 files into your frontend project:

```txt
project-root/
  CLAUDE.md
  docs/
    RULE.md
```

If you use Claude Code or Codex, place `CLAUDE.md` in the project root. If you use Cursor, you can copy the content of `RULE.md` into `.cursorrules`.

## When should you use this?

- You use Claude/Codex/Cursor for daily frontend development.
- AI agents often edit unrelated files.
- AI agents refactor or add dependencies without approval.
- Your team wants a shared AI-agent workflow before pull requests.
- Your project uses React, Next.js, Vite, TypeScript or a frontend monorepo.
